Class format

This module is divided into three blocks:

  1. Overview and building programmable game machines.
  2. The development of games and programming languages.
  3. Game engines.

Block 1: Overview and building programmable machines

In this block:

  • the metaphor of the tower block;
  • building a foundation of understanding layer by layer;
  • the practical application of this knowledge in a game context; and,
  • exercises to test your understanding.

Block 2: The development of games and programming languages

In this block:

  • development of new generations of languages;
  • applying these languages to replicate earlier designs; and,
  • exercises to test your understanding.

Block 3: Game engines

In this block:

  • development of game engines as simulators;
  • continuing the development of programming in simulators; and,
  • exercises to test your understanding.

Dyslexia and learning difficulties

  • Make sure you have contacted Disability & Dyslexia Support (DDS).
  • Programming is particularly hard on your eyes.
  • The programs we're using in this module are customisable for font and background colour.
  • Whether you have dyslexia or not, configuring your system can reduce eye strain.
  • My preferred font is Microsoft Consolas.
  • If you prefer to change the font, ensure that it clearly distinguishes the following characters:
    • 1 I l |
    • 0 O
    • ' "
    • : ;
    • . ,
    • () {} [] <>

The tower block

The programming body of knowledge:

  • Too big to cover is 36 hours.
  • 10,000 hours (5-6 years) practice become an expert.
  • Concepts build on each other.
  • There is a core of self-similar principles.

Project management

You will be expected to manage your own time:

  • Maintain a log of everything you do.
  • Estimate the duration of tasks based on previously logged times.
  • Produce schedules for planned work to keep your workload manageable.

Exercises

In most weeks you will be expected to complete exercises:

  • They should not take long.
  • You may complete them during class time - if you're quick.
  • If you get stuck, ask for help.

Source control

  • A collaborative database for source files.
  • It preserves version history.
  • It has tools to manage the merging of source files.
  • All of your files for exercises and work logs should submitted to it.
  • The source control system we are using in this module is called subversion.
  • Our server is hosted by Google Code.
  • Your introductory class will cover this system and how to use it.

Game project

You will create a game for your assessment:

  • small scope;
  • work alone or in teams;
  • build in Unity;
  • should demonstrate your understanding of programming concepts;
  • focus on custom programmed behaviour of game objects;
  • polished art, design, or audio is not required;
