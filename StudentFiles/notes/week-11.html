  2. Object oriented programming

    • what is object oriented programming?

      • not a language
      • supporting features
      • new perspective on a program
      • simulation (Simula67)
      • continuation of abstraction
      • interacting behaviours

    • classes and instances

      • factory analogy:
        • class is the template
        • instances are the products
      • in Unity:
        • a prefab is a template
        • drag and drop to make an instance
        • each script defines a class
        • variables exist per instance

    • message passing

      • loose coupling
      • ignore unwanted messages
      • names must match

    • inheritance and composition

      • IS-A and HAS-A
      • IS-A provides:
        • inheritance
        • implementation by overriding
        • common interface
        • polymorphism
        • tip: shallow inheritance
        • examples: Unity Object, Component
      • HAS-A provides:
        • implementation by delegation
        • customisation by combination
        • example: Unity Component variables

    • exceptions

      • what is an exception?
      • what happens to your script
      • tip: always check the console

    • co-routines

      • result of adding a yield instruction
      • you can yield to pause a function
      • the program continues when the called function finishes
      • if you don't yield the program continues straight away
